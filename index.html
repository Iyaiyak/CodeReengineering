Dispensable :

This refer to practices or techniques that are recognized as indicators of poor code quality (code smells) but are considered less critical or impactful and can be tolerated or dispensed with in certain contexts. For example, a small code duplication or a minor violation of a coding convention might be considered dispensable if the effort to fix it outweighs the benefits.

Change Preventer :

This refer to techniques or strategies aimed at preventing changes to code that introduce or exacerbate code smells. It might involve implementing automated checks, code reviews, or establishing coding standards and guidelines to catch and prevent changes that would introduce code smells. The focus here is on proactively avoiding the introduction of problematic code rather than simply identifying it after the fact.
